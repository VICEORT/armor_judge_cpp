# 一. 控制和传输架构：
## 1.总节点
hero_node
  ├─ 接收 image topic，缓存 FramePacket(image + stamp )
  ├─ vision::detector -> std::vector<DetectionResult>
  ├─ 为FramePacket相应帧加上bbox信息。
  ├─ 接收 /cloud_integrated保留sensor_msgs::msg::PointCloud2::SharedPtr，header.stamp，frame_id
  └─ 调用 aim_processor

（更细节：core 创建 FramePacket
detector 完成后把 std::vector<DetectionResult> 填入该帧
core 调 time_associator.pushFrame(frame_packet)）

## 2.建议的 FramePacket 结构：
struct FramePacket {
    rclcpp::Time stamp{0, 0, RCL_ROS_TIME};                         // 图像 header.stamp
    sensor_msgs::msg::Image::SharedPtr image_msg;
    cv::Mat image_mat;                         // 可选，若 core 已提前转好可直接填
    bool has_cv_mat{false};

    std::vector<DetectionResult> detections;   // detector 输出
    bool detections_ready{false};

    uint64_t frame_id = 0;                     // 内部流水号，可选
};

## 3.aim模块总调度
aim_processor
  ├─ time_associator: cloud_stamp -> matched FramePacket，需要调用TimeAssociator::match
  ├─ extrinsic_projection: cloud + matched frame -> 投影结果
  ├─ cloud_refine: 用 detection bbox 约束点云
  ├─ deviation_calc: 求偏差
  ├─ gimbal_compensation: 求补偿
  └─ aim_protocol: 输出协议

# 二.外参反投影架构
## 1.time_associator:
**核心功能**
接收 hero node / core 送来的 FramePacket
把 FramePacket 放进内部缓存
缓存项按 header.stamp 维护
给定一个点云时间戳，找最近的 FramePacket
判断最近匹配是否在 max_dt 之内
返回“匹配结果 + 诊断信息”
辅助功能(硬同步稳定后可删除)：
统计图像 stamp 间隔
统计点云 stamp 间隔
统计跨传感器偏移：img_stamp - cloud_stamp
判断是否存在固定 offset / 漂移 / 图像回跳 / 点云回跳

## 2.extrinsic_projection 的详细功能
**核心功能**
输入：
一帧点云
一个已匹配好的 FramePacket
投影参数（内参、畸变、外参）

输出：
对每个点的投影结果
对每个点的 RGB
对每个点的 armor_score
对每个点是否有效投影的标记

它负责：
lidar 点从 lidar 坐标系变到 camera 坐标系
剔除 z <= 0 的点
相机投影到 (u, v)
畸变修正
判断点是否落到图像内
从图像取 RGB
判断 (u, v) 是否在 armor bbox 内（若想更细，可以改成：bbox 内中心附近更高，边缘较低，多个 bbox 取最大值，按 detection confidence 加权）
生成中间结果结构：

**两层结果：**
A. 单点结果结构：
struct ProjectedPoint {
    float x_lidar, y_lidar, z_lidar;
    float intensity;
    float x_cam, y_cam, z_cam;
    float u, v;
    uint8_t r, g, b;
    float armor_score;     // 0~1
    int armor_box_index;   // 命中的 armor bbox，下标；未命中为 -1
    bool valid_projection; // 是否投影到图像内
};
B. 整帧输出结构
struct ProjectionResult {
    rclcpp::Time cloud_stamp;  // Timestamp from the LiDAR point cloud message
    rclcpp::Time image_stamp;  // Timestamp from the associated frame

    std::vector<ProjectedPoint> points;  // Projected 2D points, each with information about the original 3D point
    std::vector<ArmorBBox> armor_boxes;  // Detected armor bounding boxes on the image

    size_t num_input_points = 0;  // Total number of input LiDAR points
    size_t num_front_points = 0;  // Number of points in front of the camera
    size_t num_valid_projected_points = 0;  // Number of valid 2D projected points
    size_t num_armor_points = 0;  // Number of points inside armor bounding boxes

    bool ok = false;  // Flag to indicate if the projection was successful
};
